## 二、报名帖正文（可直接复制）

### 1. Demo 简介

**是什么：** NovelForge 是一款基于 Pipeline 流水线的 AI 辅助长篇网文创作 Web 工作台，集成全文记忆维护与双轨质量审计。

**面向谁：** 核心用户是长篇网文作者（日更 4000–10000 字的签约写手），他们需要持续保持角色一致性、世界观稳定性和伏笔回收能力。

**主要功能：**

1. **Pipeline 智能创作流水线**：大纲规划 → 上下文装配 → 流式正文写作 → FastAudit 快速检查 → DeepAudit 深度审计，一键生成完整章节。
   > 附截图：大纲面板 + 场景卡

2. **全文记忆系统**：自动维护角色档案、世界观设定、伏笔追踪，写作时自动注入前文上下文，确保长篇一致性。
   > 附截图：记忆面板四标签页

3. **双轨质量审计**：FastAudit（禁用词/字数/段落/标点/重复短语 5 项规则检查）+ DeepAudit（角色一致性/情节逻辑/AI味/叙事节奏/风格匹配 5 维深度评分）。
   > 附截图：审计报告面板

### 2. Demo 创作思路

**灵感来源：** 我自己是网文读者，也是 ERP 实施顾问，日常接触大量文档写作工作。网文作者最大的痛苦不是"写不出来"，而是"写着写着忘了前面埋的伏笔"和"角色性格前后不一致"。现有的 AI 写作工具大多是单次 Prompt 生成，缺乏对长篇作品的持续记忆维护。

**想解决的问题：**
- 长篇小说角色设定容易写偏（写到第 200 章时，主角性格可能和第 10 章不一致）
- 世界观设定碎片化（修炼体系、势力分布散落在各处，作者自己记不清）
- 伏笔铺了忘记回收（前期埋的悬念后期没有呼应，读者骂街）
- AI 生成内容质量不稳定（有 AI 味、节奏拖沓、不符合网文风格）

**为什么做这个方向：** 网文是一个年产值超 300 亿的内容产业，但作者工具链停留在 Word/WPS + 脑图的阶段。AI 时代应该有一套专门为长篇创作设计的工作流，而不是把 ChatGPT 当搜索引擎用。

```
┌─────────────────────────────────────────────────────────┐
│                     NovelForge v0.1.0                     │

├─────────────────────────────────────────────────────────┤
│  Memory:   角色档案 / 世界观设定 / 伏笔追踪 / 章节记忆      │
│  Audit:    FastAudit (5项规则) + DeepAudit (5维评分)       │
└─────────────────────────────────────────────────────────┘
```
